## 二、架构决策

**保持 OpenClaw，学习 Nanobot/NanoClaw**

### 2.1 为什么保持 OpenClaw？

1. ✅ **功能完整** - cron, 多通道，Skill 系统
2. ✅ **生态成熟** - 被 ScienceClaw/PaperClaw 验证
3. ✅ **已开发完成** - 切换成本高
4. ✅ **平台级应用** - 适合 Redigg 定位

### 2.2 学习 Nanobot/NanoClaw

**Nanobot 可学习**:
- ✅ 轻量级设计（~4000 行代码）
- ✅ 快速启动（2 分钟上手）
- ✅ 代码可读性（研究友好）
- ✅ 频繁迭代（每天更新）

**NanoClaw 可学习**:
- ✅ 容器化隔离（每用户一容器）
- ✅ 文件系统 IPC（简单可靠）
- ✅ Agent Swarms（多 Agent 协作）

### Phase 3（2-4 周）：容器化改造

**目标**: 提升安全性，用户隔离

**学习 NanoClaw**:
1. **容器化隔离**

   - 依赖隔离

2. **文件系统 IPC**
   - 简化 Agent-Tool 通信
   - 易于调试
   - 跨语言友好

**实现方案**:
```
Redigg Agent v2.0

OpenClaw Gateway (主框架)
    ↓
┌─────────────────────────────┐
│  Skill 执行层                │
│  ┌─────────────────────┐    │
│  │ Docker Container 1  │    │
│  │ (用户隔离)          │    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ Docker Container 2  │    │
│  │ (用户隔离)          │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

**具体任务**:
- [ ] 研究 NanoClaw 容器化方案
