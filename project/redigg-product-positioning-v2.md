## 🌟 核心功能

### 1. 持续搜索 (Continuous Search)

**传统模式**:
```
用户搜索 → 返回结果 → 结束
(每次从零开始)
```

**Redigg 模式**:
```
用户设定主题 → 持续监控 → 新论文自动推送 → 积累到记忆树
(7×24 小时不间断)
```

**使用场景**:
- 📌 "帮我跟踪大模型推理方向的最新进展"
- 📌 "持续关注金融 NLP 领域的论文"
- 📌 "监控量子计算在药物发现中的应用"

**技术实现**:
- arXiv API 定时轮询（每日/每周）
- Semantic Scholar Alerts
- Google Scholar Alerts 集成
- 基于 PageIndex 树的自动分类

---

### 2. 概念串联 (Concept Linking)

**传统模式**:
```
论文 A → 孤立结果
论文 B → 孤立结果
论文 C → 孤立结果
(用户自己发现关联)
```

**Redigg 模式**:
```
论文 A ─┬─→ 概念 X ←─┬─→ 论文 B
        │            │
        └─→ 方法 Y ←─┘
              ↓
        发现：A 和 B 都用了方法 Y 解决不同问题
        ↓
        生成洞察：方法 Y 可能适用于更多场景
```

**使用场景**:
- 🔗 "帮我找出这 3 篇论文的共同点"
- 🔗 "这个方法在哪些领域被应用过？"
- 🔗 "这些研究之间有什么联系？"

**技术实现**:
- PageIndex 树形记忆结构
- 概念提取（LLM）
- 跨节点推理（多步检索）
- 知识图谱可视化

---

### 3. 长期工作 (Long-term Work)

**传统模式**:
```
会话 1: 搜索 A → 结束（无记忆）
会话 2: 搜索 B → 结束（无记忆）
会话 3: 搜索 C → 结束（无记忆）
(每次都是新的开始)
```

**Redigg 模式**:
```
会话 1: 搜索 A → 存储到记忆树
会话 2: 搜索 B → 关联到 A → 生成洞察
会话 3: 搜索 C → 关联到 A+B → 形成理论
(持续积累，越用越聪明)
```

**使用场景**:
- 📚 博士研究（3-5 年持续跟踪）
- 📚 长期项目（数月到数年）
- 📚 跨领域研究（整合多个领域知识）

**技术实现**:
- PageIndex 树形记忆（永久存储）
- 记忆巩固机制（Working → Short → Long）
- 定期回顾和更新（主动推送"去年今天"）

### 持续搜索流程

```
1. 用户设定主题
   ↓
2. 创建监控任务（每日/每周）
   ↓
3. 定时搜索（arXiv/Semantic Scholar）
   ↓
4. 新论文自动分类到记忆树
   ↓
5. 与现有记忆关联（概念串联）
   ↓
6. 生成洞察和推送
   ↓
7. 用户确认 → 永久存储
```

### 概念串联引擎

```typescript
interface ConceptLink {
  concept: string;
  papers: string[]; // 涉及该概念的论文
  methods: string[]; // 使用的方法
  relationships: Relationship[];
}

interface Relationship {
  type: 'similar_method' | 'different_problem' | 'extension' | 'contradiction';
  paperA: string;
  paperB: string;
  description: string;
  confidence: number;
}

async function discoverConceptLinks(
  tree: MemoryTree
): Promise<ConceptLink[]> {
  // Step 1: 提取所有节点的概念
  const concepts = await extractConcepts(tree);
  
  // Step 2: 跨节点查找关联
  const links = await findCrossNodeLinks(concepts);
  
  // Step 3: 生成洞察
  const insights = await generateInsights(links);
  
  return insights;
}
```
