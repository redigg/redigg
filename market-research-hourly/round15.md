## 🔍 竞品动态推断（基于已知信息）

### 主要竞品状态

**Elicit**:
- 优势：文献搜索和摘要功能成熟
- 劣势：云端 SaaS，隐私担忧
- 推断：短期内不太可能有重大架构变化
- Redigg 机会：本地优先 + 隐私保护

**SciSpace**:
- 优势：PDF 解析和解释功能强
- 劣势：单点工具，缺乏端到端流程
- 推断：继续深耕 PDF 理解
- Redigg 机会：端到端研究自动化

**Litmaps**:
- 优势：引用图谱可视化
- 劣势：需要手动操作，自动化程度低
- 推断：可能增加 AI 功能
- Redigg 机会：自动化引用图谱生成

**Wordware** ($30M 融资):
- 优势：资金充足，团队扩大
- 劣势：通用工作流平台，非研究专用
- 推断：需要时间将资金转化为产品
- Redigg 机会：时间窗口约 6-12 个月

**CrewAI** (10 万 + 开发者):
- 优势：生态成熟，Agent 框架完善
- 劣势：编程导向，非研究导向
- Redigg 机会：可作为技术依托，专注研究场景

## 🎯 种子用户策略

### 目标用户画像

**核心用户**:
- 博士研究生（理工科为主）
- 年轻教授/研究员
- 企业研发人员
- 独立研究者

**痛点**:
1. 文献综述耗时（平均 2-4 周）
2. 容易遗漏关键论文
3. 难以追踪最新研究
4. 协作困难（团队研究）
5. 可复现性问题

### 获取策略

**阶段 1：信任背书（10-20 人）**
- Vix 提供初始联系人
- 熟人推荐，接受率高
- 深度访谈，收集反馈

**阶段 2：学术网络扩展（50-100 人）**
- 种子用户推荐
- 学术社交媒体（ResearchGate 等）
- 大学实验室合作

**阶段 3：公开测试（500+ 人）**
- Landing Page 发布
- 社交媒体营销
- 学术社区推广

### Onboarding 流程

```
用户注册 → 欢迎邮件 → 首次任务引导 → 
完成第一个文献综述 → 收集反馈 → 迭代优化
```

**关键指标**:
- 注册转化率 > 30%
- 首次任务完成率 > 70%
- 用户满意度 > 4/5
- 推荐率 > 40%

## 📋 Phase 2 测试计划

### TaskPlanner 测试场景

**场景 1：文献综述任务**
```
输入："量子计算在密码学中的应用"
期望步骤：
1. arxiv_search(query="quantum computing cryptography")
2. paper_summarizer(papers=[...])
3. citation_graph(papers=[...])
4. synthesis(summaries=[...], graph=...)
输出：结构化文献综述
```

**场景 2：实验设计任务**
```
输入："设计一个验证 X 假设的实验"
期望步骤：
1. literature_review(hypothesis="X")
2. hypothesis(formalize="X")
3. experimental_design(hypothesis="X")
4. metrics(design=...)
输出：完整实验方案
```

**场景 3：数据分析任务**
```
输入："分析这个数据集并找出规律"
期望步骤：
1. data_loading(path="...")
2. cleaning(data=...)
3. analysis(data=...)
4. visualization(results=...)
5. interpretation(visuals=...)
输出：数据分析报告
```

### QualityChecker 测试标准

**5 个质量维度**:

| 维度 | 权重 | 标准 | 测量方法 |
|------|------|------|----------|
| 完整性 | 25% | 覆盖率 > 90% | 关键论文不遗漏 |
| 准确性 | 30% | 引用正确率 > 95% | 人工抽查 |
| 相关性 | 20% | 内容相关度 > 4/5 | LLM 评估 |
| 格式 | 15% | 符合学术规范 | 规则检查 |
| 引用质量 | 10% | 来源可靠性 > 4/5 | 期刊/会议评级 |

**通过标准**:
- 综合得分 > 4/5
- 无严重错误（引用错误、事实错误）
- 用户满意度 > 4/5
