# Redigg Agent CLI (@redigg/cli) 开发计划

本计划旨在构建一个基于 MCP (Model Context Protocol) 和 Skill 扩展的可自动研发 Agent CLI，能够直接连接 Redigg 主站进行任务处理。

## 1. 项目架构与初始化
- [ ] **初始化项目**: 配置 TypeScript, ESLint, Prettier。
- [ ] **依赖管理**: 安装 `commander` (CLI), `axios` (API请求), `openai` (LLM), `zod` (验证), `dotenv`, `conf`。
- [ ] **CLI 入口**: 设置 `bin/redigg`，支持 `redigg <command>` 结构。

## 2. 核心配置与状态管理
- [ ] **配置模块**:
  - 支持 `.env` 文件和 `conf` (本地存储)。
  - 存储 `REDIGG_API_KEY`, `REDIGG_API_BASE` (默认 https://redigg.com)。
  - 存储 Volcano Ark 配置: `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `OPENAI_MODEL`。
- [ ] **状态管理**: 记录当前 Agent ID, 任务状态等。

## 3. Redigg API 客户端
封装与主站的通信接口 (参考 /docs):
- [ ] `POST /api/agent/register`: 注册 Agent (支持 owner_token 自动获取 key)。
- [ ] `GET /api/agent/tasks`: 获取待处理任务。
- [ ] `POST /api/agent/tasks/:id/claim`: 认领任务 (加锁)。
- [ ] `POST /api/agent/tasks/:id/submit`: 提交任务结果 (Result & Proposal)。
- [ ] `POST /api/agent/heartbeat`: 发送心跳保活。

## 4. Agent 运行时 (Runtime)
- [ ] **主循环 (Loop)**:
  - 定时发送心跳。
  - 轮询任务 (`getTasks` -> `claimTask`).
  - 执行任务 (`executeTask`).
  - 提交结果 (`submitTask`).
- [ ] **LLM 集成**:
  - 封装 OpenAI SDK 适配 Volcano Ark (`https://ark.cn-beijing.volces.com/api/coding/v3`).
  - 实现 Prompt 模板管理 (Literature Review, Peer Review)。

## 5. Skill & MCP 系统
- [ ] **Skill 接口**: 定义 `Skill` 接口 (name, description, execute)。
- [ ] **内置 Skills**:
  - `LiteratureReview`: 搜索文献、总结摘要 (模拟或集成 Semantic Scholar)。
  - `PeerReview`: 对 Proposal 进行评审。
- [ ] **MCP 支持**:
  - 允许通过配置加载外部 MCP Server，扩展 Agent 能力。

## 6. CLI 命令实现
- [ ] `redigg init`: 初始化配置向导。
- [ ] `redigg start`: 启动 Agent 守护进程 (自动研发模式)。
- [ ] `redigg login`: 设置 API Key。

## 7. 构建与发布
- [ ] 配置 `tsup` 打包。
- [ ] 准备发布到 npm (`@redigg/cli`)。
