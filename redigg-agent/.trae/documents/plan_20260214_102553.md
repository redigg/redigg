# Redigg Agent CLI 进度回顾与待办计划

根据 `https://redigg.com/docs` 和当前代码状态，以下是完成度分析及后续计划。

## ✅ 已完成 (Completed)
1.  **基础架构 (Infrastructure)**
    - [x] CLI 骨架 (`commander`, `init`, `config`, `start`).
    - [x] 自动注册流程 (`register` with owner token).
    - [x] API 客户端 (封装了 `axios` 请求主站 API).
    - [x] **SSE 长连接**: 实现了 `/api/events` 监听，替代了 Webhook，支持实时接收任务。
    - [x] 心跳保活 (`heartbeat`).

2.  **核心循环 (Core Loop)**
    - [x] 任务获取 -> 认领 (Lock) -> 执行 -> 提交 (Submit).
    - [x] LLM 集成 (Volcano Ark / OpenAI 适配器).

3.  **扩展性 (Extensibility)**
    - [x] Skill 注册表框架.
    - [x] MCP Client 集成 (支持连接外部 MCP Server).

4.  **真实的科研 Skills (Priority: High)**
    - **Literature Review**:
      - [x] 集成 Semantic Scholar API 查找真实论文。
      - [x] 实现论文摘要提取与综合 (Synthesis)。
      - [x] 生成符合 Redigg 格式的 Markdown 报告 (包含引用)。
    - **Peer Review**:
      - [x] 实现 `PeerReviewSkill`，用于评审其他 Agent 提交的 Proposal。
      - [x] 实现评审逻辑：检查 Novelty, Methodology, Clarity 等维度。
      - [x] 自动提取决策结果：Accept/Minor Revision/Major Revision/Reject。

5.  **动态任务分发 (Priority: High)**
    - [x] 根据 `task.type` (如 `literature_review` vs `peer_review`) 动态选择 Skill。
    - [x] 处理 Skill 未找到的情况 (Fallback 或报错)。
    - [x] 支持多种任务类型：literature_review, peer_review, experiment_design, data_analysis。

6.  **增强 LLM 能力 (Priority: Medium)**
    - [x] 优化 Prompt 模板，使其生成的科研内容更专业。
    - [x] 支持 Structured Output 以便更好地解析 LLM 结果。
    - [x] 实现决策自动提取功能。
